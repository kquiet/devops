# HorizontalPodAutoscaler (HPA)
 - K8S內建API Resource
 - 用途
    - 自動調整Pod副本數：根據CPU、記憶體或自定義指標
 - 使用自定義指標需先在GKE Cluster內安裝Google Cloud提供的[Custom Metrics Stackdriver Adapter](https://github.com/GoogleCloudPlatform/k8s-stackdriver/tree/master/custom-metrics-stackdriver-adapter)，讓HPA可以取得Google Cloud Managed Service for Prometheus的metrics
```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: ocr-gpu
  namespace: wormhole2
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: ocr-gpu
  minReplicas: 1
  maxReplicas: 8
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 70
  - type: Pods
    pods:
      metric:
        name: prometheus.googleapis.com|DCGM_FI_PROF_GR_ENGINE_ACTIVE|gauge
      target:
        type: AverageValue
        averageValue: 70
  behavior:
    scaleDown:
      policies:
      - periodSeconds: 30
        type: Percent
        value: 25
      stabilizationWindowSeconds: 300
    scaleUp:
      policies:
      - periodSeconds: 15
        type: Pods
        value: 2
      - periodSeconds: 15
        type: Percent
        value: 50
      selectPolicy: Max
      stabilizationWindowSeconds: 0
```

# PodDisruptionBudget (PDB)
 - K8S內建API Resource
 - 用途
    - 限制非預期中斷(如升級或重啟)的Pod數量
    - 非預期中斷
       - Node Upgrade / Drain
       - 手動刪除Pod
       - Node Pool Scale-down
```yaml
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: ocr-gpu
  namespace: wormhole2
spec:
  maxUnavailable: 1
  selector:
    matchLabels:
      app: ocr-gpu
```
